# TG 随机视频机器人

一个简单的 Telegram 机器人，自动记录频道中的视频，用户发送命令即可随机转发一条视频。

## 功能

- 🎥 **自动收录** - 频道中每发布一个新视频，机器人自动记录
- 🎲 **随机转发** - 发送 `/random` 随机获取一条历史视频
- 📊 **统计查看** - 发送 `/stats` 查看已收录的视频数量

## 快速开始

### 1. 创建 Telegram 机器人

1. 在 Telegram 中搜索 [@BotFather](https://t.me/BotFather)
2. 发送 `/newbot` 创建一个新机器人，按提示设置名称和用户名
3. 创建成功后，你会得到一个 **Token**（格式如 `123456:ABC-DEF...`），复制保存

### 2. 将机器人设为频道管理员

1. 进入你的频道
2. 点击频道名称 → "管理员" → "添加管理员"
3. 搜索你刚创建的机器人并添加
4. **权限至少需要**：✅ 发布消息（机器人转发时需要）

### 3. 配置并运行

```bash
# 1. 克隆/进入项目目录
cd tg随机视频

# 2. 安装依赖
pip install -r requirements.txt

# 3. 配置环境变量
cp .env.example .env
# 编辑 .env，填入你的 BOT_TOKEN 和 CHANNEL_ID

# 4. 启动机器人
python bot.py
```

### 4. 使用机器人

在 Telegram 中搜索你的机器人，发送以下命令：

| 命令 | 功能 |
|------|------|
| `/start` | 开始使用，查看帮助 |
| `/random` | 随机转发一条频道视频 |
| `/stats` | 查看已收录的视频数量 |

## 注意事项

- 机器人必须先设为频道管理员，才能监听到频道的视频发布
- 只有机器人成为管理员**之后**发布的视频才会被收录，之前的视频不会自动同步
- 视频数据存储在本地 `videos.db` SQLite 文件中，更换设备时记得迁移
- 建议使用 **screen** 或 **systemd** 保持机器人在服务器上长期运行

## 长期运行（可选）

Linux/Mac 上使用 screen：

```bash
screen -S tgbot
python bot.py
# 按 Ctrl+A 再按 D 退出 screen，机器人继续在后台运行
```
